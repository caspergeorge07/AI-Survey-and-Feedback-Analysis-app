## Purpose

This document defines starting design-system guidelines for SurveyIQ. It should support a professional SaaS experience focused on clarity, trust, and executive-ready insight.

## 1. Design Principles

- Professional and calm.
- Insight-led, not decorative.
- Dense enough for business users, but not cluttered.
- Clear hierarchy between summary, evidence, and actions.
- Consistent interactions across upload, analysis, and reporting.

## 2. Visual Tone

SurveyIQ should feel:

- Trustworthy.
- Analytical.
- Polished.
- Efficient.
- Executive-ready.

It should not feel:

- Playful or overly casual.
- Overly technical.
- Visually noisy.
- Like a generic AI chat tool.

## 3. Color System

Starting palette:

| Token | Use | Example |
|---|---|---|
| Primary | Main actions, section accents | Deep teal |
| Ink | Primary text | Near-black |
| Muted | Secondary text | Blue gray |
| Surface | Cards and panels | White |
| Background | App background | Light gray |
| Positive | Positive sentiment | Green |
| Neutral | Neutral sentiment | Amber |
| Negative | Negative sentiment | Red |

Exact color tokens should be finalized during visual design.

## 4. Typography

Starting guidance:

- Use a clean sans-serif font.
- Prioritize readability in tables and reports.
- Use clear heading hierarchy.
- Avoid oversized hero typography inside the authenticated app.

## 5. Layout

Authenticated app layout should use:

- Persistent navigation.
- Clear page titles.
- Compact filters.
- Full-width working areas.
- Tables for dense data.
- Cards for repeated summary objects.

## 6. Components

Core components:

- Buttons.
- File upload.
- Select controls.
- Data tables.
- Metric cards.
- Sentiment badges.
- Theme tags.
- Progress indicators.
- Download actions.
- Empty states.
- Error states.

## 7. Data Visualization

Initial chart types:

- Horizontal bar charts for theme counts.
- Horizontal or stacked bars for sentiment counts.
- Trend lines for future time-series views.

Guidelines:

- Always label counts clearly.
- Avoid chart decoration that reduces readability.
- Use consistent sentiment colors.
- Include sample size where relevant.

## 8. Content Style

Voice:

- Clear.
- Concise.
- Business-oriented.
- Evidence-led.

Avoid:

- Overpromising certainty.
- Vague AI language.
- Jargon where plain language works.

## 9. Accessibility

Starting requirements:

- Sufficient color contrast.
- Keyboard-accessible controls.
- Visible focus states.
- Descriptive button labels.
- Charts supported by text summaries.
