## Purpose

This document outlines the product roadmap for SurveyIQ. It separates current MVP capabilities from near-term improvements and future SaaS capabilities.

## 1. Roadmap Principles

- Preserve the working analysis workflow.
- Improve insight quality before adding commercial complexity.
- Keep exports reliable.
- Add SaaS capabilities only after the core workflow is validated.
- Avoid unnecessary features that do not support survey feedback intelligence.

## 2. Current MVP

Implemented or planned in current MVP scope:

- CSV and Excel upload.
- First 10-row preview.
- Detected column display.
- Feedback column selection.
- OpenAI-powered response analysis.
- Batched analysis for larger files.
- Canonical theme consolidation.
- Sentiment counts.
- Theme counts.
- Analysed CSV download.
- Management PDF report generation.
- Local file handling.

## 3. Near-Term Priorities

Placeholder priorities:

1. Improve frontend access to PDF report download.
2. Add clearer processing states for longer analyses.
3. Add test coverage for backend analysis and reporting.
4. Add file size and row count guidance.
5. Improve error messaging for failed analysis stages.

## 4. SaaS Foundation

Future foundation areas:

- Authentication.
- Workspaces.
- Projects.
- Reports library.
- User roles.
- Billing and usage.
- Background job processing.
- Persistent storage.

## 5. Future Enterprise

Future enterprise areas:

- SSO.
- Data governance.
- Custom taxonomy.
- Advanced permissions.
- Audit logs.
- Benchmarking.

## 6. Not Yet Planned

The following are intentionally not part of the current MVP:

- Payments.
- Subscriptions.
- Teams.
- Enterprise governance.
- Database-backed collaboration.
